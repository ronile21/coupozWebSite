Accessibility Statement

CoupoZ is committed to making its website accessible to as many people as possible, regardless of technology or ability.

Last updated: 23 March 2026

Conformance status

We aim to conform to the Web Content Accessibility Guidelines (WCAG) 2.2, Level AA . The current status is partial conformance: most criteria are met, but some areas — noted below — have known limitations or could not be fully verified without live assistive-technology testing.

Technical specifications

Accessibility of this website relies on the following technologies:

  • HTML5 (semantic landmarks, native interactive elements)
  • CSS (including prefers-reduced-motion and focus-visible styles)
  • JavaScript (ARIA state management, keyboard handling)

These technologies are relied upon for conformance. The website has been tested with:

  • Keyboard-only navigation
  • Chrome and Firefox on desktop with screen magnification
  • Automated audits using axe-core (WCAG 2.2 AA ruleset)
  • VoiceOver on macOS (Safari) — manual spot checks

What we have done

  • Added a Skip to main content link as the first focusable element on every page.
  • Used semantic HTML landmarks — header, nav, main, footer, section — throughout.
  • Ensured one clear h1 per page with a consistent heading hierarchy.
  • Provided accessible names for all interactive controls (buttons, links) using visible text or aria-label.
  • Applied visible focus-visible outlines on all focusable elements.
  • Managed aria-expanded and aria-controls on the FAQ accordion and mobile navigation menu.
  • Hidden collapsed FAQ answer panels from assistive technology using aria-hidden when closed.
  • Added a aria-live region to announce language changes to screen readers.
  • Marked all decorative SVG icons as aria-hidden="true".
  • Fixed colour contrast for body text (section subtitles, metric labels) to meet WCAG AA ratios.
  • Respected prefers-reduced-motion by disabling scroll-reveal animations and card hover transforms for users who request reduced motion.
  • Ensured the lang attribute on <html> is updated dynamically when the language is switched (English / Thai).
  • Used native <button> and <a> elements for all interactive controls — no div click handlers.
  • Added an in-house accessibility options panel — a floating button (bottom-left) that opens a small display-and-motion preferences panel. Options include text size adjustment, high contrast, link underlining, reduced motion, comfortable text spacing, and an alternative readable font. Settings are saved in localStorage, can be reset at any time, and do not require sign-in. The panel is fully keyboard accessible with a focus trap and Escape key support.

Known limitations

  • Thai language screen-reader support: Thai content is served with the correct lang="th" attribute, but synthesis quality depends on the user's screen reader and its Thai voice support. We have not been able to perform comprehensive Thai AT testing.
  • Tailwind CSS via CDN: The site currently loads Tailwind from a content delivery network. This introduces a small render-blocking dependency that we intend to resolve by switching to a local build in a future release.
  • Decorative inline illustrations: The platform illustration SVG in the Solution section has an aria-label describing it as a generic illustration. It carries no essential information; screen-reader users receive all equivalent information from the surrounding text.
  • Contact channel buttons (coming soon): WhatsApp, Telegram, and Line contact options are not yet active. They are displayed as non-interactive elements with visible "Coming soon" labels.

What could not be verified automatically

  • Cognitive accessibility and plain-language readability beyond automated checks.
  • Touch target sizing on all physical mobile devices.
  • Full JAWS and NVDA compatibility on Windows — only axe-core and VoiceOver checks have been performed.

Feedback and contact

If you encounter an accessibility barrier or have suggestions to help us improve, we want to hear from you. Please contact us:

  • Email: info@coupoz.com (please include "Accessibility feedback" in the subject line)

We aim to respond to accessibility feedback within 5 business days.

Formal complaints

If you are not satisfied with our response to an accessibility concern, you may contact the relevant national authority or equality body in your jurisdiction. In Thailand, this includes the Department of Empowerment of Persons with Disabilities (DEP) .
